# errors.md — エラーコード辞書

> ASP3カーネルのサービスコールが返す `ER` 型エラーコード（メインエラーコード）の一覧。  
> AIがテスト失敗・実行時エラーを解釈するための辞書。  
> **数値は `include/itron.h` / `include/kernel.h` の定義を正とする**（下表はμITRON4.0標準の慣用値）。

---

## メインエラーコード一覧

| コード | 値 | 意味 | 典型的な原因 |
|---|---|---|---|
| `E_OK` | 0 | 正常終了 | — |
| `E_SYS` | -5 | システムエラー | カーネル内部の異常・回復不能状態 |
| `E_NOSPT` | -9 | 未サポート機能 | 当該構成で無効化された機能を呼んだ |
| `E_RSFN` | -10 | 予約機能コード | 不正な機能コード |
| `E_RSATR` | -11 | 予約属性 | オブジェクト属性に不正なビット指定 |
| `E_PAR` | -17 | パラメータエラー | 引数の値が範囲外・不正 |
| `E_ID` | -18 | 不正ID番号 | 存在しない/範囲外のオブジェクトID |
| `E_CTX` | -25 | コンテキストエラー | 呼び出してはいけない文脈で実行（例：割込みハンドラ内で待ち系API） |
| `E_MACV` | -26 | メモリアクセス違反 | 保護されたメモリへのアクセス |
| `E_OACV` | -27 | オブジェクトアクセス違反 | アクセス権のないオブジェクト操作 |
| `E_ILUSE` | -28 | サービスコール不正使用 | 使用条件を満たさない呼び出し |
| `E_NOMEM` | -33 | メモリ不足 | 必要なメモリが確保できない |
| `E_NOID` | -34 | ID番号不足 | 動的生成でID枯渇 |
| `E_NORES` | -35 | 資源不足 | ID以外の資源が不足 |
| `E_OBJ` | -41 | オブジェクト状態エラー | オブジェクトが操作可能な状態にない |
| `E_NOEXS` | -42 | オブジェクト未生成 | 対象オブジェクトが存在しない |
| `E_QOVR` | -43 | キューイングオーバーフロー | カウント/キューの上限超過 |
| `E_RLWAI` | -49 | 待ち状態の強制解除 | `rel_wai` 等で待ちが解除された |
| `E_TMOUT` | -50 | ポーリング失敗 / タイムアウト | 指定時間内に条件成立せず |
| `E_DLT` | -51 | 待ちオブジェクトの削除 | 待ち中にオブジェクトが削除された |
| `E_CLS` | -52 | 待ちオブジェクトの状態変化 | 待ち中に待ち対象状態が変化した |
| `E_WBLK` | -57 | ノンブロッキング受付 | 非待ち呼び出しが受け付けられた |
| `E_BOVR` | -58 | バッファオーバーフロー | バッファ容量を超えた |

## AIへの解釈ヒント

| 観測した状況 | 疑うべきエラー | 確認先 |
|---|---|---|
| 起動直後に停止 | `E_SYS` | カーネル初期化・スタック設定 |
| API呼び出しが `-18` を返す | `E_ID` | オブジェクトIDの定義（.cfg）と使用箇所 |
| API呼び出しが `-25` を返す | `E_CTX` | 割込みハンドラ内で待ち系APIを呼んでいないか |
| `wai_sem` 等が `-50` を返す | `E_TMOUT` | タイムアウト値・先行する `sig_sem` の有無 |
| `wai_sem` 等が `-49` を返す | `E_RLWAI` | `ter_tsk` / `rel_wai` による解除 |
| 構成エラーでビルド失敗 | `E_RSATR` / `E_PAR` | .cfg の属性・パラメータ記述 |

## ログ上のエラー表記

構造化ログ（`docs` の構造化ログ仕様）では以下の形式で出力される：

```
T=<tick>,EV=ERR,API=<service_call>,CODE=<value>
```

例：
```
T=2000,EV=ERR,API=wai_sem,CODE=-50    # → E_TMOUT（タイムアウト）
T=2500,EV=ERR,API=act_tsk,CODE=-18    # → E_ID（不正ID）
```

`scripts/parse_slog.py` はCODE値を本辞書の意味に変換して出力できる。

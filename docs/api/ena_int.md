# ena_int — 割込みの許可

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

> **ターゲット定義機能。** `ena_int` がサポートされるかどうかはターゲット依存
> （ターゲット定義マクロ `TOPPERS_SUPPORT_ENA_INT` が定義されている場合のみ提供される）。
> サポートされないターゲットでは呼び出せない。

## C言語API
```c
ER ercd = ena_int(INTNO intno);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `intno` | `INTNO` | 許可対象の割込み番号（プロセッサ依存） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_PAR` | `intno` が割込み番号として不正・範囲外 |
| `E_OBJ` | `intno` が `CFG_INT` で設定されていない割込み番号 |

> 本サービスコールがサポートされていないターゲットでは、呼び出せない（プロトタイプが提供されない）。

## 機能
`intno` で指定した割込み番号の割込みを許可する。

割込みを許可すると、その割込み番号に対応する割込み要求が受け付けられるようになる。
`dis_int` で禁止した割込みを再び許可する場合に使う。

`intno` は `CFG_INT` で設定されている割込み番号でなければならない。設定されていない
割込み番号を指定すると `E_OBJ` を返す。`CFG_INT` で `TA_ENAINT` 属性を指定しなかった
割込みは初期状態で禁止されており、`ena_int` で許可する必要がある。

非タスクコンテキストから呼び出す場合は、同一機能の `iena_int` を使ってもよい
（ASP3では `iena_int` は `ena_int` のマクロとして定義されている）。

## 使用例
```c
void critical_section(void) {
    dis_int(INTNO_UART);    /* UART割込みを禁止 */
    /* 割込みと競合する処理 */
    ena_int(INTNO_UART);    /* UART割込みを許可 */
}
```

## 関連
- `dis_int` — 割込みの禁止（対操作）
- `clr_int` / `ras_int` / `prb_int` — 割込み要求のクリア / 要求 / チェック（ターゲット定義機能）
- [CFG_INT](CFG_INT.md) — 割込み要求ラインの設定（`TA_ENAINT` で初期許可状態を指定）
- 仕様書 4.9 割込み管理機能

# DEF_ICS — 非タスクコンテキスト用スタック領域の定義（静的API）

## 種別
〔S〕静的API（システムコンフィギュレーションファイル `.cfg` に記述）

> **構造の正本は `kernel/kernel_api.def`（cfgが読むapi-table）。** パラメータ列・型はそこで宣言される。
> 意味・エラー・使用例の正本は [統合仕様書 4.9 割込み管理機能](https://www.toppers.jp/docs/tech/tgki_spec-370/tgki_spec-370.html)。
> 本ファイルは両者をまとめた人間/AI向けリファレンス。

## 静的API
```
DEF_ICS({ istksz, istk });
```

## api-table 定義（構造の正本）
`kernel/kernel_api.def` の該当行：
```
DEF_ICS { .istksz &istk? }
```

| 記法 | 意味 |
|---|---|
| `.istksz` | 符号無し整数定数式（スタックサイズ） |
| `&istk?` | 一般整数定数式・オプション（スタック領域先頭番地） |

> `DEF_ICS` はキーパラメータ（`#`）を持たない。システム全体で高々1回だけ記述する。

## パラメータ
| 位置 | 名称 | C-API型 | 説明 |
|---|---|---|---|
| パケット | `istksz` | `SIZE` | 非タスクコンテキスト用スタックのサイズ（バイト） |
| パケット | `istk` | `STK_T*` | 非タスクコンテキスト用スタック領域の先頭番地（省略可。`NULL`=自動割付け） |

## エラーコード（コンフィギュレータが検出）
| コード | 条件 |
|---|---|
| `E_PAR` | `istksz` が下限未満・不正 |
| `E_NOMEM` | スタック領域が確保できない（自動割付け時） |
| `E_OBJ` | `DEF_ICS` が複数回記述された |

> 静的APIは返値を持たない。エラーはコンフィギュレータが報告する。

## 機能
非タスクコンテキスト（割込みハンドラ・割込みサービスルーチン・CPU例外ハンドラ・タイム
イベントハンドラなど）が使用するスタック領域を定義する。

`istk` を省略（または `NULL`）すると、コンフィギュレータが `istksz` バイトのスタックを
自動割付けする。`istk` を指定する場合は、`istksz` バイトのスタック領域の先頭番地を渡す。

`DEF_ICS` を記述しない場合は、ターゲット定義のデフォルトの非タスクコンテキスト用スタック
サイズが用いられる。`DEF_ICS` はシステム全体で高々1回だけ記述できる。

## 使用例
```
/* 非タスクコンテキスト用スタックを 4096 バイト、自動割付けで確保 */
DEF_ICS({ 4096, NULL });
```

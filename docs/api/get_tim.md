# get_tim — システム時刻の参照

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

## C言語API
```c
ER ercd = get_tim(SYSTIM *p_systim);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `p_systim` | `SYSTIM *` | システム時刻の格納先（`SYSTIM` = `uint64_t`） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |
| `*p_systim` | `SYSTIM` | 現在のシステム時刻（ミリ秒） |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX`〔s〕 | CPUロック状態で呼び出した |

## 機能
カーネルが管理するシステム時刻を `*p_systim` に格納して返す。

`SYSTIM` は `uint64_t` であり、単位は**ミリ秒**。カーネル起動からの経過時間を表す。
オーバーフローするまでの時間は約5.8億年であり、実用上オーバーフローを考慮しなくてよい。

システム時刻はシステムタイマ割込み（カーネルティック）のたびに更新される。
精度はシステムタイマの分解能（通常 1ms）に依存する。

より高い分解能（マイクロ秒）が必要な場合は `fch_hrt` を使う。
性能評価用のプロセッサ時間（タスク実行時間）を取得する場合は `get_utm` を使う。

## 使用例
```c
void benchmark_task(intptr_t exinf) {
    SYSTIM t0, t1;

    get_tim(&t0);
    do_heavy_work();
    get_tim(&t1);

    syslog(LOG_INFO, "elapsed: %llu ms", (unsigned long long)(t1 - t0));
}
```

## 関連
- `fch_hrt` — 高分解能タイマの参照（マイクロ秒単位）
- `get_utm` — 性能評価用システム時刻の参照（プロセッサ時間）

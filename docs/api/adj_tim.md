# adj_tim — システム時刻の調整

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

## C言語API
```c
ER ercd = adj_tim(int32_t adjtim);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `adjtim` | `int32_t` | システム時刻の調整時間（マイクロ秒）。正で進め、負で遅らせる |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | CPUロック状態で呼び出した |
| `E_PAR` | `adjtim` が範囲外（`TMIN_ADJTIM`（-1,000,000）以上 `TMAX_ADJTIM`（1,000,000）以下でない） |
| `E_OBJ` | 調整によりシステム時刻が逆転するなど、時刻の調整が許されない状態であった |

## 機能
カーネルが管理するシステム時刻を `adjtim` マイクロ秒だけ調整する。
`adjtim` が正の値ならシステム時刻を進め、負の値なら遅らせる。

1回の呼び出しで調整できる範囲は `TMIN_ADJTIM`（-1,000,000μs）以上
`TMAX_ADJTIM`（1,000,000μs）以下である。これを超える調整は複数回に分けて行う。

`adj_tim` は内部のイベント時刻（`current_evttim`）を直接調整するため、`set_tim` と異なり、
登録済みのタイムイベント（周期通知・アラーム通知・タイムアウト等）の発生タイミングにも影響する。
時刻を進めると、それまでに発生時刻に達したタイムイベントが発生する。

調整によってシステム時刻が過去に登録されたタイムイベントの処理順序を壊すなど、
時刻の調整が許されない状態である場合は `E_OBJ` を返し、システム時刻は変更されない。

時刻の連続性を保つ必要がなく、システム時刻を即座に任意の値へ設定したい場合は `set_tim` を使う。

## 使用例
```c
void time_sync_task(intptr_t exinf) {
    int32_t diff_us;

    diff_us = get_reference_time_diff_us();   /* 外部時刻源との差（μs） */

    /* 1回あたりの調整上限を超えないようにクリップして補正 */
    if (diff_us > TMAX_ADJTIM)      diff_us = TMAX_ADJTIM;
    else if (diff_us < TMIN_ADJTIM) diff_us = TMIN_ADJTIM;

    adj_tim(diff_us);
}
```

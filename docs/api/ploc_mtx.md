# ploc_mtx — ミューテックスのロック（ポーリング）

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態でも `E_CTX`）

## C言語API
```c
ER ercd = ploc_mtx(ID mtxid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mtxid` | `ID` | 対象ミューテックスのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（ミューテックスをロックした） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態（`CHECK_TSKCTX_UNL`） |
| `E_ID` | `mtxid` が不正・範囲外 |
| `E_ILUSE` | 優先度上限ミューテックス（`TA_CEILING`）で、自タスクのベース優先度が上限優先度より低い |
| `E_OBJ` | 自タスクが既に対象ミューテックスをロックしている（多重ロック） |
| `E_TMOUT` | 他タスクがロック中で、ロックを獲得できなかった |

## 機能
`mtxid` で指定したミューテックスのロックを試みる。`loc_mtx` のポーリング版で、
ロックできない場合に待ち状態に遷移せず、即座に `E_TMOUT` でリターンする点が異なる。

対象ミューテックスがどのタスクにもロックされていない場合は、自タスクがロックを獲得して
`E_OK` を返す。既に他タスクがロックしている場合は `E_TMOUT` を返す。

ASP3のミューテックスは**優先度上限プロトコル（`TA_CEILING`）**のみをサポートする。
自タスクのベース優先度が上限優先度より低い場合は、ロックできず `E_ILUSE` を返す。

## 使用例
```c
void worker(intptr_t exinf) {
    if (ploc_mtx(SHARED_MTX) == E_OK) {
        /* クリティカルセクション */
        unl_mtx(SHARED_MTX);
    }
    else {
        /* ロックできなかった場合の処理 */
    }
}
```

## 関連
- `loc_mtx` — ロック（待ちあり）
- `tloc_mtx` — ロック（タイムアウト付き）
- `unl_mtx` — ミューテックスのロック解除
- `CRE_MTX` — ミューテックスの生成（静的API）

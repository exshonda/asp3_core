# sns_loc — CPUロック状態の参照

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能（CPUロック状態でも呼出し可能）

## C言語API
```c
bool_t state = sns_loc(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `state` | `bool_t` | CPUロック状態なら `true`、CPUロック解除状態なら `false` |

## エラーコード
なし。

> この関数は `bool_t`（真偽値）を返し、`ER` 型のエラーコードを返さない。
> どのようなコンテキスト・状態（CPUロック状態を含む）からでも呼び出せ、エラーにならない。

## 機能
処理単位がCPUロック状態か否かを参照する。CPUロック状態であれば `true`、
CPUロック解除状態であれば `false` を返す。

状態を参照するだけで、カーネルの状態を変化させない。CPUロック状態であっても
呼び出すことができ、エラーにならない。

## 使用例
```c
void maybe_unlock(void) {
    if (sns_loc()) {
        unl_cpu();   /* CPUロック中なら解除 */
    }
}
```

## 関連
- `loc_cpu` / `unl_cpu` — CPUロック状態への遷移 / 解除
- `sns_ctx` — コンテキストの参照
- `sns_dpn` — ディスパッチ保留状態の参照

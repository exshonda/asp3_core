# ena_dsp — ディスパッチの許可

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = ena_dsp(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した、またはCPUロック状態で呼び出した（`CHECK_TSKCTX_UNL`） |

## 機能
ディスパッチ禁止状態を解除し、ディスパッチ許可状態へ遷移させる。これにより
保留されていたディスパッチが実行され、必要であれば自タスクは他タスクに
横取りされる。

ASP3では、**ディスパッチ禁止状態でないときに `ena_dsp` を呼び出してもエラーに
ならず、ディスパッチ許可状態のまま `E_OK` を返す**。`dis_dsp` と `ena_dsp` の
対応は**ネストしない**（μITRON4.0と異なり、ネストカウントを持たない）。
ディスパッチ禁止状態で `dis_dsp` を何回呼んでいても、1回の `ena_dsp` で
ディスパッチ許可状態に戻る。

ディスパッチ許可状態への遷移により最高優先順位のタスクが切り替わる場合は、
この呼び出しの中でディスパッチが起こる。なお `ena_dsp` の実行で自タスクに
終了要求（`ras_ter`）が保留されていた場合は、ここでタスク終了処理が起動される。

## 使用例
```c
void task_local_critical(intptr_t exinf) {
    dis_dsp();
    update_task_shared_data();
    ena_dsp();   /* 保留されていたディスパッチがここで起こりうる */
}
```

- `sns_dsp` — ディスパッチ禁止状態の参照

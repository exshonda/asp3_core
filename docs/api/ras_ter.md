# ras_ter — タスクの終了要求

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = ras_ter(ID tskid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskid` | `ID` | 終了要求の対象タスクのID番号（自タスクは指定不可） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態で呼び出した |
| `E_ID` | `tskid` が不正・範囲外 |
| `E_ILUSE` | `tskid` に自タスクを指定した |
| `E_OBJ` | 対象タスクが休止状態（DORMANT）である |

## 機能
`tskid` で指定したタスクに対して終了要求を出す。

対象タスクが終了を許可している（`dis_ter` による終了禁止状態でない）場合は、対象タスクを
ただちに終了させ、休止状態（DORMANT）に遷移させる。対象タスクが何らかの待ち状態にあった
場合は、その待ちを解除したうえで終了させる。

対象タスクが終了を禁止している（`dis_ter` で終了禁止状態にしている）場合は、終了要求フラグ
（`raster`）をセットするにとどめる。このとき、対象タスクが待ち状態にあれば待ちを強制解除し
（待ち中のサービスコールに `E_RASTER` を返させる）、実行可能状態に遷移させる。
対象タスクが `ena_ter` で終了を許可した時点で、セットされた終了要求にしたがって終了する。

`ras_ter` は自タスクを対象とできない（`E_ILUSE`）。`TSK_SELF` も指定できない。

## 使用例
```c
void main_task(EXINF exinf) {
    ER ercd = ras_ter(WORKER_TASK);
    if (ercd != E_OK) {
        syslog(LOG_ERROR, "ras_ter failed: %d", ercd);
    }
}
```

## 関連
- `ter_tsk` — タスクの強制終了（終了禁止状態を無視せず即時終了させるものではない点に注意）
- `dis_ter` / `ena_ter` — タスク終了の禁止 / 許可
- `sns_ter` — タスク終了禁止状態の参照
- 仕様書 4.3 タスク終了機能

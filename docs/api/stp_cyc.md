# stp_cyc — 周期通知の動作停止

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼び出すと `E_CTX`）

## C言語API
```c
ER ercd = stp_cyc(ID cycid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `cycid` | `ID` | 対象周期通知のID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態で呼び出した |
| `E_ID` | `cycid` が不正・範囲外 |

## 機能
`cycid` で指定した周期通知を動作停止状態にする。

動作停止状態にすると、登録済みの次回起動予定が解除され、以降は通知ハンドラが起動されなくなる。

対象周期通知が既に動作停止状態である場合でも、エラーとはならず `E_OK` を返す（何もしない）。

停止した周期通知を再び動作開始するには `sta_cyc` を呼び出す。このとき起動位相時間は
`sta_cyc` を呼び出した時点を起点として計算され、停止前の位相は引き継がれない。

## 使用例
```c
void shutdown_task(intptr_t exinf) {
    stp_cyc(SAMPLE_CYC);     /* 周期サンプリングを停止 */
}
```

## 関連
- `sta_cyc` — 周期通知の動作開始
- `ref_cyc` — 周期通知の状態参照

# prcv_pdq — 優先度データキュー受信（ポーリング）

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = prcv_pdq(ID pdqid, intptr_t *p_data, PRI *p_datapri);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `pdqid` | `ID` | 対象優先度データキューのID番号 |
| `p_data` | `intptr_t *` | 受信データの格納先 |
| `p_datapri` | `PRI *` | 受信データの優先度の格納先 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |
| `*p_data` | `intptr_t` | 受信したデータ |
| `*p_datapri` | `PRI` | 受信したデータの優先度 |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（データを受信した） |
| `E_CTX` | 非タスクコンテキスト／CPUロック状態 |
| `E_ID` | `pdqid` が不正・範囲外 |
| `E_TMOUT` | キューが空でデータを受信できなかった（ポーリング失敗） |

## 機能
`pdqid` で指定した優先度データキューから、最も優先度の高いデータを1件取り出し、`*p_data` に
そのデータを、`*p_datapri` にそのデータの優先度を格納する。`rcv_pdq` のポーリング版であり、
待ち状態に遷移しない。同一優先度のデータが複数ある場合は、先に送信されたデータが取り出される
（FIFO順）。

キューにデータが存在する場合は即座にリターンする。データを取り出したことで送信待ちタスクが
いる場合は、そのタスクが送信したデータをキューに追加してから待ち解除する。

キューが空で送信待ちタスクもいない場合は、待たずに `E_TMOUT` を返す。

## 使用例
```c
void consumer_task(intptr_t exinf) {
    intptr_t data;
    PRI datapri;
    ER ercd = prcv_pdq(JOB_PDQ, &data, &datapri);
    if (ercd == E_OK) {
        process_job((Job *)data, datapri);
    } else {
        do_other_work();   /* データが無ければ別処理 */
    }
}
```

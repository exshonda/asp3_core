# ter_tsk — タスクの強制終了

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = ter_tsk(ID tskid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskid` | `ID` | 強制終了の対象タスクのID番号（自タスクは指定不可） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態で呼び出した |
| `E_ID` | `tskid` が不正・範囲外 |
| `E_ILUSE` | `tskid` に自タスクを指定した |
| `E_OBJ` | 対象タスクが休止状態（DORMANT）である |

## 機能
`tskid` で指定したタスクを強制的に終了させ、休止状態（DORMANT）に遷移させる。
`dis_ter` による終了禁止状態にかかわらず即座に終了させる点で、`ras_ter`（終了要求）とは
異なる。

対象タスクが待ち状態にあった場合は、その待ちを解除したうえで終了させる。終了対象タスクに
対する起動要求がキューイングされている場合は、終了処理後にそのタスクを再度起動する
（キューイング数を1減じて実行可能状態に遷移させる）。

`ter_tsk` は自タスクを対象とできない（`E_ILUSE`）。`TSK_SELF` も指定できない。
自タスクを終了させる場合は `ext_tsk` を使う。

## 使用例
```c
void supervisor_task(EXINF exinf) {
    ER ercd = ter_tsk(WORKER_TASK);
    if (ercd != E_OK) {
        syslog(LOG_ERROR, "ter_tsk failed: %d", ercd);
    }
}
```

## 関連
- `ext_tsk` — 自タスクの終了
- `ras_ter` — タスクの終了要求（終了禁止状態を尊重する）
- `act_tsk` — タスクの起動（DORMANT から READY へ）

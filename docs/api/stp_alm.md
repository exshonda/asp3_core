# stp_alm — アラーム通知の動作停止

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

## C言語API
```c
ER ercd = stp_alm(ID almid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `almid` | `ID` | 対象アラームのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | CPUロック状態で呼び出した |
| `E_ID` | `almid` が不正・範囲外 |

## 機能
`almid` で指定したアラーム通知を動作停止状態にする。

動作停止状態にすると、登録済みの起動予定が解除され、設定された起動時刻に達してもアラームハンドラは
起動されなくなる。タイムアウト処理などで、設定したアラームを期限前にキャンセルする用途に用いる。

対象アラームが既に動作停止状態である場合でも、エラーとはならず `E_OK` を返す（何もしない）。

非タスクコンテキストから呼び出すための別名 `istp_alm(almid)` が定義されている（実体は `stp_alm` と同じ）。

## 使用例
```c
void command_task(intptr_t exinf) {
    FLGPTN ptn;

    send_command();
    sta_alm(CMD_TIMEOUT_ALM, 100000U);   /* 100ms タイムアウトを設定 */

    wai_flg(SYS_FLG, EV_RESPONSE | EV_TIMEOUT, TWF_ORW, &ptn);
    stp_alm(CMD_TIMEOUT_ALM);            /* 応答が先に来たらタイムアウトをキャンセル */
}
```

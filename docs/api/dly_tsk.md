# dly_tsk — 時間経過待ち

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = dly_tsk(RELTIM dlytim);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `dlytim` | `RELTIM` | 待ち時間（マイクロ秒、相対時間） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（指定時間が経過した） |
| `E_PAR` | `dlytim` が `TMAX_RELTIM`（4,000,000,000）を超える |
| `E_CTX` | 非タスクコンテキスト／CPUロック／ディスパッチ禁止状態 |
| `E_RLWAI` | 待ち状態が強制解除された（`rel_wai`） |
| `E_RASTER` | 待ち状態の間に自タスクへの終了要求（`ras_ter`）があった |

## 機能
自タスクを `dlytim` マイクロ秒の間、時間経過待ち状態に遷移させる。指定した以上の
時間が経過すると、待ち状態が解除されて `E_OK` を返す。

`wup_tsk`（`tslp_tsk`の解除）とは異なり、`dly_tsk` の待ちは起床要求では解除されない。
`dlytim` に 0 を指定した場合も待ち状態に遷移する（高分解能タイマの次のカウントアップで
解除される。`tslp_tsk(TMO_POL)` がポーリングで待ち状態に入らないのとは対照的）。

## 使用例
```c
void periodic_worker(intptr_t exinf) {
    while (true) {
        do_work();
        dly_tsk(100000U);   /* 100ms周期 */
    }
}
```

## 関連
- `tslp_tsk` — 起床待ち（タイムアウト付き、`wup_tsk`で解除可能）
- 周期通知（`sta_cyc`）— より正確な周期実行
- 仕様書 4.2 タスク付属同期機能

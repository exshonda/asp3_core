# sig_sem — セマフォ資源の返却

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

## C言語API
```c
ER ercd = sig_sem(ID semid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `semid` | `ID` | 対象セマフォのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_ID` | `semid` が不正・範囲外 |
| `E_CTX`〔s〕 | CPUロック状態で呼び出した |
| `E_NOEXS`〔D〕 | 対象セマフォが未登録（動的生成対応カーネル） |
| `E_QOVR` | セマフォ資源数が上限（`maxsem`）を超えた |

## 機能
`semid` で指定したセマフォに資源を1つ返却する。

セマフォの待ち行列にタスクが存在する場合は、先頭タスクを待ち解除して実行可能状態（READY）に
遷移させる（資源数は増加しない）。

待ち行列が空の場合は、資源数を1増やす。資源数がセマフォ生成時に指定した最大値（`maxsem`）を
超える場合は `E_QOVR` を返す。

割込みハンドラや周期通知ハンドラから呼び出すことができる（〔TI〕）。

## 使用例
```c
/* 割込みハンドラから通知 */
void uart_rx_isr(void) {
    receive_byte_to_buffer();
    sig_sem(UART_SEM);      /* 受信タスクを起こす */
}

void uart_rx_task(intptr_t exinf) {
    while (true) {
        wai_sem(UART_SEM);
        process_received_data();
    }
}
```

- `twai_sem` — 資源の獲得（タイムアウト付き）

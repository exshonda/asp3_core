# can_wup — タスク起床要求のキャンセル

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`）

## C言語API
```c
ER_UINT wupcnt = can_wup(ID tskid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskid` | `ID` | 対象タスクのID番号。`TSK_SELF`（=0）で自タスク指定 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `wupcnt` | `ER_UINT` | 正の値またはエラーコード。正常時はキャンセルした起床要求のキューイング数（0 または 1） |

## エラーコード
| コード | 条件 |
|---|---|
| `0` または `1` | 正常終了。キャンセルした起床要求のキューイング数（`E_OK` 相当の正常値） |
| `E_CTX` | 非タスクコンテキスト／CPUロック状態（`CHECK_TSKCTX_UNL`） |
| `E_ID` | `tskid` が不正・範囲外（`TSK_SELF` 以外で `VALID_TSKID` 不成立） |
| `E_OBJ` | 対象タスクが休止状態（DORMANT） |

## 機能
`tskid` で指定したタスクにキューイングされている起床要求をすべて取り消し、
取り消した起床要求のキューイング数を返す。

ASP3では起床要求のキューイング数は最大1であるため、戻り値は `0`（キューイングなし）
または `1`（1回分をキャンセルした）のいずれかとなる。実行後、対象タスクの起床要求
キューイング数は 0 になる。

戻り値は `ER_UINT` 型であり、`0` 以上が正常値（キューイング数）、負値はエラーコードである。
判定の際は `wupcnt < 0` でエラーを検出する。

`tskid` に `TSK_SELF`（=0）を指定すると自タスクを対象とする。

## 使用例
```c
void cleanup_task(intptr_t exinf) {
    ER_UINT wupcnt;

    /* 保留中の起床要求を破棄してから待ちに入る */
    wupcnt = can_wup(TSK_SELF);
    if (wupcnt >= 0 && wupcnt > 0) {
        /* 起床要求が残っていた */
    }
    slp_tsk();
}
```

## 関連
- `wup_tsk` — タスクの起床（起床要求をキューイングする）
- `slp_tsk` / `tslp_tsk` — 起床待ち（キューイングされた起床要求を消費する）
- 仕様書 4.2 タスク付属同期機能

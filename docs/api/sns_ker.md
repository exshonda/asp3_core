# sns_ker — カーネル非動作状態の参照

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能（CPUロック状態でも、カーネル非動作状態でも呼出し可能）

## C言語API
```c
bool_t state = sns_ker(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `state` | `bool_t` | カーネル非動作状態なら `true`、カーネル動作状態なら `false` |

## エラーコード
なし。

> この関数は `bool_t`（真偽値）を返し、`ER` 型のエラーコードを返さない。
> どのようなコンテキスト・状態（CPUロック状態・カーネル非動作状態を含む）からでも呼び出せ、エラーにならない。

## 機能
カーネルが非動作状態か否かを参照する。カーネル非動作状態であれば `true`、
カーネル動作状態であれば `false` を返す。実装上は内部フラグ `kerflg`
（カーネル動作中フラグ）の否定を返す。

**カーネル非動作状態**とは、カーネルの初期化が完了する前（`sta_ker` 前）、または
カーネルの終了処理（`ext_ker`）後の状態を指す。この状態ではほとんどの
サービスコールが正しく動作しないため、それらを呼び出す前に
カーネルが動作中かどうかを判定する用途で用いる。

状態を参照するだけで、カーネルの状態を変化させない。カーネル非動作状態を
含むあらゆる状態から呼び出すことができ、エラーにならない。

## 使用例
```c
void safe_log(const char *msg) {
    if (!sns_ker()) {
        syslog(LOG_NOTICE, msg);   /* カーネル動作中のみログ出力 */
    }
}
```

- `ext_ker` — カーネルの終了
- `sns_ctx` — コンテキストの参照

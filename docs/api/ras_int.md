# ras_int — 割込みの要求

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

> **ターゲット定義機能。** `ras_int` がサポートされるかどうかはターゲット依存
> （ターゲット定義マクロ `TOPPERS_SUPPORT_RAS_INT` が定義されている場合のみ提供される）。
> サポートされないターゲットでは呼び出せない。

## C言語API
```c
ER ercd = ras_int(INTNO intno);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `intno` | `INTNO` | 割込み要求を発生させる割込み番号（プロセッサ依存） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_PAR` | `intno` が割込み番号として不正・範囲外 |
| `E_OBJ` | `intno` が `CFG_INT` で設定されていない、または割込み要求を発生できない割込み番号 |

> 本サービスコールがサポートされていないターゲットでは、呼び出せない（プロトタイプが提供されない）。

## 機能
`intno` で指定した割込み番号に対して、ソフトウェアから割込み要求を発生させる。

割込みコントローラに対してソフトウェア的に割込み要求をセットし、割込みを擬似的に
発生させたい場合に使う。

`intno` は `CFG_INT` で設定されており、かつ割込み要求の発生が可能な割込み番号で
なければならない。条件を満たさない場合は `E_OBJ` を返す
（カーネル内部で `check_intno_cfg` と `check_intno_raise` の両方を確認する）。

非タスクコンテキストから呼び出す場合は、同一機能の `iras_int` を使ってもよい
（ASP3では `iras_int` は `ras_int` のマクロとして定義されている）。

## 使用例
```c
void trigger_soft_int(void) {
    ras_int(INTNO_SOFT);    /* ソフトウェア割込みを要求 */
}
```

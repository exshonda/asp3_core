# ini_dtq — データキューの再初期化

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = ini_dtq(ID dtqid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `dtqid` | `ID` | 対象データキューのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキスト／CPUロック状態 |
| `E_ID` | `dtqid` が不正・範囲外 |

## 機能
`dtqid` で指定したデータキューを初期状態に戻す。

具体的には、送信待ち行列と受信待ち行列につながれていたタスクの待ちをすべて解除し
（解除されたタスクには `E_DLT` が返される）、データキュー管理領域に格納されていたデータを
すべて破棄する（格納データ数を0に戻す）。

待ちを解除されたタスクが自タスクより高い優先度を持つ場合、本サービスコールからのリターン時に
ディスパッチが発生する。

## 使用例
```c
void reset_handler(intptr_t exinf) {
    /* キューを空にし、待ちタスクを解除して系を初期化 */
    ini_dtq(MSG_DTQ);
}
```

## 関連
- `snd_dtq` — データキュー送信（ブロッキング）
- `rcv_dtq` — データキュー受信（ブロッキング）
- `ref_dtq` — データキュー状態の参照
- `CRE_DTQ` — データキューの生成（静的API）

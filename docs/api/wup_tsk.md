# wup_tsk — タスクの起床

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

## C言語API
```c
ER ercd = wup_tsk(ID tskid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskid` | `ID` | 起床対象タスクのID番号（`TSK_SELF` 不可） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_ID` | `tskid` が不正・範囲外 |
| `E_CTX`〔s〕 | CPUロック状態で呼び出した |
| `E_NOEXS`〔D〕 | 対象タスクが未登録（動的生成対応カーネル） |
| `E_OBJ` | 対象タスクが休止状態（DORMANT）または自タスク指定（非タスクでの `TSK_SELF`） |
| `E_QOVR` | 起床要求キューイング数が上限（1回）を超えた |

## 機能
`tskid` で指定したタスクを起床させる。対象タスクが `slp_tsk`（または `tslp_tsk`）による
起床待ち状態の場合は、そのタスクを実行可能状態（READY）に遷移させる。

対象タスクが起床待ち状態でない場合は、起床要求を1回分キューイングする。
次に `slp_tsk` / `tslp_tsk` を呼び出したとき、待ちに入らず即座にリターンする。
起床要求のキューイングは1回分のみ許容されており、既にキューイング済みの場合は `E_QOVR`。

`dly_tsk` による時間経過待ち中のタスクは起床できない（`rel_wai` を使う）。

`tskid` に `TSK_SELF`（＝0）は指定できない。

## 使用例
```c
void button_isr(void) {
    /* 割込みハンドラからタスクを起床する */
    wup_tsk(BUTTON_TASK);
}

void button_task(intptr_t exinf) {
    while (true) {
        slp_tsk();      /* 起床待ち */
        handle_button();
    }
}
```

## 関連
- `slp_tsk` / `tslp_tsk` — 起床待ち（`wup_tsk` で解除される）
- `can_wup` — キューイングされた起床要求のキャンセル
- `rel_wai` — 待ち状態の強制解除（`dly_tsk` 等の非起床待ちにも使える）
- 仕様書 4.2 タスク付属同期機能

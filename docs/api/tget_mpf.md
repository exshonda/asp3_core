# tget_mpf — 固定長メモリブロックの獲得（タイムアウト付き）

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態・ディスパッチ禁止状態でも `E_CTX`）

## C言語API
```c
void *blk;
ER ercd = tget_mpf(ID mpfid, void **p_blk, TMO tmout);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mpfid` | `ID` | 対象固定長メモリプールのID番号 |
| `p_blk` | `void **` | 獲得したメモリブロックの先頭番地を格納する変数へのポインタ |
| `tmout` | `TMO` | タイムアウト時間（μs）。`TMO_POL`（=0）でポーリング、`TMO_FEVR`（=-1）で永久待ち |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |
| `*p_blk` | `void *` | 獲得したメモリブロックの先頭番地（`E_OK` 時のみ有効） |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（メモリブロックを獲得した） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態／ディスパッチ禁止状態（`CHECK_DISPATCH`） |
| `E_ID` | `mpfid` が不正・範囲外 |
| `E_PAR` | `tmout` が不正（`VALID_TMOUT` 違反） |
| `E_TMOUT` | タイムアウト時間内にメモリブロックを獲得できなかった（`TMO_POL` 指定で空きなしを含む） |
| `E_RASTER` | 自タスクに対してタスク終了要求がある（`ras_ter`） |
| `E_RLWAI` | 待ち状態が強制解除された（`rel_wai`） |

## 機能
`mpfid` で指定した固定長メモリプールから、`tmout` で指定した時間を上限としてメモリブロックを
獲得する。`get_mpf` のタイムアウト付き版である。

割付け可能なメモリブロックがあれば即座に獲得して先頭番地を `*p_blk` に格納し `E_OK` を返す。
ない場合は、メモリブロックが返却されるか `tmout` で指定した時間が経過するまで待ち状態に
遷移する。時間内に獲得できなかった場合は `E_TMOUT` を返す。`tmout` に `TMO_POL` を指定すると
ポーリング（`pget_mpf` 相当）、`TMO_FEVR` を指定すると永久待ち（`get_mpf` 相当）となる。

## 使用例
```c
void worker(intptr_t exinf) {
    void *blk;
    ER ercd = tget_mpf(WORK_MPF, &blk, 1000);   /* 最大1ms待つ */
    if (ercd == E_OK) {
        /* blk を使った処理 */
        rel_mpf(WORK_MPF, blk);
    }
    else if (ercd == E_TMOUT) {
        /* タイムアウト処理 */
    }
}
```

## 関連
- `get_mpf` — 獲得（永久待ち）
- `pget_mpf` — 獲得（ポーリング）
- `rel_mpf` — 固定長メモリブロックの返却
- `CRE_MPF` — 固定長メモリプールの生成（静的API）

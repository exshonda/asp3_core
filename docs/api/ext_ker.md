# ext_ker — カーネルの終了

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能（CPUロック状態でも呼出し可能）

## C言語API
```c
ER ercd = ext_ker(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常に終了処理へ移行した場合は復帰しない。復帰した場合は `E_SYS`（システムエラー） |

## エラーコード
| コード | 条件 |
|---|---|
| `E_SYS` | 終了処理から戻ってしまった場合（通常は到達しない） |

> 正常時はこの呼び出しから戻らない（カーネルの終了処理に移行する）。

## 機能
カーネルを終了させる。`ext_ker` を呼び出すと、まず割込みロック状態に移行し、
カーネル動作中フラグ（`kerflg`）を偽にしてカーネル動作を終了する。その後、
非タスクコンテキストに切り換えてカーネル終了処理（`exit_kernel`）を呼び出す。

カーネル終了処理では、構成ファイルに登録された終了処理ルーチン（`ATT_TER` で
登録）が登録順に実行され、最後にターゲット依存の終了処理（`target_exit`）が
呼び出される。

正常に終了処理へ移行した場合、`ext_ker` の呼び出し元には**戻らない**。
何らかの理由で終了処理から戻った場合に限り `E_SYS` を返すが、これは通常
到達しない経路である（コンパイラの警告対策）。

`ext_ker` はCPUロック状態・割込みロック状態・ディスパッチ禁止状態を含む
任意の状態から呼び出せる。

## 使用例
```c
void shutdown_task(intptr_t exinf) {
    syslog(LOG_NOTICE, "shutting down kernel");
    ext_ker();   /* ここから戻らない */
}
```

# pget_mpf — 固定長メモリブロックの獲得（ポーリング）

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態でも `E_CTX`）

## C言語API
```c
void *blk;
ER ercd = pget_mpf(ID mpfid, void **p_blk);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mpfid` | `ID` | 対象固定長メモリプールのID番号 |
| `p_blk` | `void **` | 獲得したメモリブロックの先頭番地を格納する変数へのポインタ |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |
| `*p_blk` | `void *` | 獲得したメモリブロックの先頭番地（`E_OK` 時のみ有効） |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（メモリブロックを獲得した） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態（`CHECK_TSKCTX_UNL`） |
| `E_ID` | `mpfid` が不正・範囲外 |
| `E_TMOUT` | 割付け可能なメモリブロックがなく、獲得できなかった |

## 機能
`mpfid` で指定した固定長メモリプールからのメモリブロック獲得を試みる。`get_mpf` の
ポーリング版で、割付け可能なメモリブロックがない場合に待ち状態に遷移せず、即座に
`E_TMOUT` でリターンする点が異なる。

割付け可能なメモリブロックがある場合は、それを獲得して先頭番地を `*p_blk` に格納し
`E_OK` を返す。ない場合は `E_TMOUT` を返す。

## 使用例
```c
void worker(intptr_t exinf) {
    void *blk;
    if (pget_mpf(WORK_MPF, &blk) == E_OK) {
        /* blk を使った処理 */
        rel_mpf(WORK_MPF, blk);
    }
    else {
        /* メモリ不足時の処理 */
    }
}
```

## 関連
- `get_mpf` — 獲得（待ちあり）
- `tget_mpf` — 獲得（タイムアウト付き）
- `rel_mpf` — 固定長メモリブロックの返却
- `CRE_MPF` — 固定長メモリプールの生成（静的API）

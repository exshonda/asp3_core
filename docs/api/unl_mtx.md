# unl_mtx — ミューテックスのロック解除

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態でも `E_CTX`）

## C言語API
```c
ER ercd = unl_mtx(ID mtxid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mtxid` | `ID` | 対象ミューテックスのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（ロックを解除した） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態（`CHECK_TSKCTX_UNL`） |
| `E_ID` | `mtxid` が不正・範囲外 |
| `E_OBJ` | 自タスクが最後にロックしたミューテックスではない（自タスクがロックしていない、またはロック順序に反する） |

## 機能
`mtxid` で指定したミューテックスのロックを解除する。

解除できるのは、**自タスクが最後にロックしたミューテックス**のみである。ASP3では
タスクがロックしているミューテックスはロックした順序の逆順（後入れ先出し）で解除する
必要があり、それ以外を指定すると `E_OBJ` を返す（`p_mtxcb != p_runtsk->p_lastmtx`）。

ロックを解除すると、待ち行列の先頭タスク（最も優先度が高く、同優先度では最も古い）が
ロックを獲得し待ち解除される。優先度上限ミューテックス（`TA_CEILING`）の解除に伴い、
自タスクの現在優先度は再計算され、必要に応じて引き下げられる（優先度上昇状態の解消）。
これによりディスパッチが必要になればディスパッチが発生する。

## 使用例
```c
void worker(intptr_t exinf) {
    if (loc_mtx(SHARED_MTX) == E_OK) {
        /* クリティカルセクション */
        unl_mtx(SHARED_MTX);
    }
}
```

## 関連
- `loc_mtx` — ロック（永久待ち）
- `ploc_mtx` / `tloc_mtx` — ロック（ポーリング／タイムアウト付き）
- `ini_mtx` — ミューテックスの再初期化（強制的なロック解除）

# prb_int — 割込み要求のチェック

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

> **ターゲット定義機能。** `prb_int` がサポートされるかどうかはターゲット依存
> （ターゲット定義マクロ `TOPPERS_SUPPORT_PRB_INT` が定義されている場合のみ提供される）。
> サポートされないターゲットでは呼び出せない。

## C言語API
```c
ER_BOOL ercd = prb_int(INTNO intno);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `intno` | `INTNO` | 割込み要求の有無を調べる割込み番号（プロセッサ依存） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER_BOOL` | 正常時は真偽値（要求あり=`true` / 要求なし=`false`）、異常時はエラーコード（負値） |

> `ER_BOOL` は「エラーコードまたは真偽値」を表す型（`int_t`）。返値が負値ならエラーコード、
> そうでなければ割込み要求の有無を示す真偽値（`true` / `false`）として解釈する。

## エラーコード
| コード | 条件 |
|---|---|
| `E_PAR` | `intno` が割込み番号として不正・範囲外 |
| `E_OBJ` | `intno` が `CFG_INT` で設定されていない割込み番号 |

> 正常終了時は `E_OK` ではなく真偽値（`true` / `false`）を返す。

> 本サービスコールがサポートされていないターゲットでは、呼び出せない（プロトタイプが提供されない）。

## 機能
`intno` で指定した割込み番号に対して、割込み要求が発生しているか（保留されているか）を
チェックする。

割込み要求が発生していれば `true`、発生していなければ `false` を返す。返値が負値の
場合はエラーコードである。

`intno` は `CFG_INT` で設定されている割込み番号でなければならない。設定されていない
割込み番号を指定すると `E_OBJ` を返す。

非タスクコンテキストから呼び出す場合は、同一機能の `iprb_int` を使ってもよい
（ASP3では `iprb_int` は `prb_int` のマクロとして定義されている）。

## 使用例
```c
void poll_uart(void) {
    ER_BOOL ercd = prb_int(INTNO_UART);
    if (ercd < 0) {
        syslog(LOG_ERROR, "prb_int failed: %d", ercd);
    }
    else if ((bool_t) ercd) {
        /* UART割込み要求が発生している */
        clr_int(INTNO_UART);
    }
}
```

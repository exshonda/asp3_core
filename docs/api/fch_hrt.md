# fch_hrt — 高分解能タイマの参照

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能（CPUロック状態・割込みハンドラ内を含む任意の状態から呼出し可能）

## C言語API
```c
HRTCNT hrtcnt = fch_hrt(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `hrtcnt` | `HRTCNT` | 高分解能タイマの現在のカウント値（マイクロ秒、`HRTCNT` = `uint32_t`） |

## エラーコード
なし（`fch_hrt` は `ER` 型ではなく `HRTCNT` 型を返す。エラーは返さない）

## 機能
高分解能タイマの現在のカウント値を取得して返す。戻り値は `ER` 型ではなく `HRTCNT` 型である点に注意する。

`HRTCNT` は `uint32_t`（既定）であり、単位は**マイクロ秒**。カウント値は高分解能タイマの
周期（`HRTCNT_BOUND` で規定される範囲）で巡回するため、得られる値は単調増加ではなく、
いずれ巻き戻る（ラップアラウンドする）。二点間の経過時間を求める場合は、巡回を考慮して
`HRTCNT` 同士の符号無し減算で差を取る。

任意の状態（CPUロック状態・割込みハンドラ内を含む）から呼び出せるように、SILの全割込みロック
状態の制御機能を用いて排他制御を実現している。このため、コンテキストエラーやCPUロックによる
エラーは発生しない。

設定可能なシステム時刻（64ビット・マイクロ秒）が必要な場合は `get_tim` を使う。

## 使用例
```c
void latency_task(intptr_t exinf) {
    HRTCNT t0, t1;

    t0 = fch_hrt();
    do_critical_section();
    t1 = fch_hrt();

    /* 符号無し減算で巡回（ラップアラウンド）を吸収して経過マイクロ秒を得る */
    syslog(LOG_INFO, "elapsed: %u us", (unsigned int)(t1 - t0));
}
```

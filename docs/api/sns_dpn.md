# sns_dpn — ディスパッチ保留状態の参照

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能（CPUロック状態でも呼出し可能）

## C言語API
```c
bool_t state = sns_dpn(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `state` | `bool_t` | ディスパッチ保留状態なら `true`、そうでなければ `false` |

## エラーコード
なし。

> この関数は `bool_t`（真偽値）を返し、`ER` 型のエラーコードを返さない。
> どのようなコンテキスト・状態（CPUロック状態を含む）からでも呼び出せ、エラーにならない。

## 機能
処理単位がディスパッチ保留状態か否かを参照する。ディスパッチ保留状態であれば
`true`、そうでなければ `false` を返す。

**ディスパッチ保留状態**とは、ディスパッチが起こらない以下のいずれかの状態を指す。

- 非タスクコンテキストの実行中（`sense_context()`）
- CPUロック状態（`sense_lock()`）
- ディスパッチ禁止状態（内部フラグ `dspflg` が偽）

すなわち `sns_ctx`・`sns_loc`・`sns_dsp` のいずれかが真であればディスパッチ保留状態と
なる。状態を参照するだけで、カーネルの状態を変化させない。CPUロック状態であっても
呼び出すことができ、エラーにならない。

## 使用例
```c
void check_can_block(void) {
    if (sns_dpn()) {
        /* ディスパッチ保留状態：待ち状態に入るAPIは呼べない */
        return;
    }
    slp_tsk();
}
```

## 関連
- `sns_ctx` — コンテキストの参照
- `sns_loc` — CPUロック状態の参照
- `sns_dsp` — ディスパッチ禁止状態の参照
- `xsns_dpn` — CPU例外発生時のディスパッチ保留状態の参照

# rel_wai — 待ち状態の強制解除

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

## C言語API
```c
ER ercd = rel_wai(ID tskid);
ER ercd = irel_wai(ID tskid);   /* 非タスクコンテキスト用の別名（実体は同一） */
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskid` | `ID` | 待ち解除対象タスクのID番号（`TSK_SELF` 不可） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（対象タスクの待ち状態を解除した） |
| `E_CTX` | CPUロック状態で呼び出した（`CHECK_UNL`） |
| `E_ID` | `tskid` が不正・範囲外（`VALID_TSKID` 不成立。`TSK_SELF` も不可） |
| `E_OBJ` | 対象タスクが待ち状態（WAITING）でない |

## 機能
`tskid` で指定したタスクが待ち状態（WAITING）にある場合、その待ち状態を強制的に解除する。
待ちを解除されたタスクは、待ち状態に入る原因となったサービスコール（`slp_tsk` / `tslp_tsk` /
`dly_tsk` / `wai_sem` / `wai_flg` など）から `E_RLWAI` を返して戻る。

`slp_tsk` の起床待ちだけでなく、`dly_tsk` の時間経過待ちや各同期・通信オブジェクトの待ちも
含めて、あらゆる待ち状態を解除できる点が `wup_tsk` と異なる。二重待ち状態（待ち状態かつ
強制待ち状態）の場合は待ち要因のみが解除され、強制待ち状態は残る。

対象タスクが待ち状態でない（実行可能状態・強制待ち状態のみ・休止状態など）場合は `E_OBJ` を返す。

待ち解除によって対象タスクの方が実行すべき優先度になった場合は、ディスパッチが起こる
（非タスクコンテキストからの呼出しでは割込み出口での遅延ディスパッチが要求される）。

`tskid` に `TSK_SELF`（＝0）は指定できない（自タスクは待ち状態にないため）。

## 使用例
```c
void watchdog_isr(void) {
    /* 応答がないワーカタスクの待ちを強制解除する */
    irel_wai(WORKER_TASK);
}

void worker_task(intptr_t exinf) {
    ER ercd = wai_sem(DATA_SEM);
    if (ercd == E_RLWAI) {
        /* rel_wai により待ちを解除された */
        recover();
    }
}
```

## 関連
- `wup_tsk` — タスクの起床（起床待ちのみを解除）
- `ter_tsk` — タスクの強制終了（待ち中も含めて終了させる）
- `slp_tsk` / `tslp_tsk` / `dly_tsk` — 解除対象となる待ち状態に入るサービスコール
- 仕様書 4.2 タスク付属同期機能

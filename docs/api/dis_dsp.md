# dis_dsp — ディスパッチの禁止

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = dis_dsp(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した、またはCPUロック状態で呼び出した（`CHECK_TSKCTX_UNL`） |

## 機能
ディスパッチ禁止状態へ遷移させる。ディスパッチ禁止状態では、タスクの切り替え
（ディスパッチ）が起こらなくなるため、自タスクは他タスクに横取りされずに
実行を続ける。割込みは許可されたままなので、割込みハンドラは実行される。

ASP3では、**既にディスパッチ禁止状態のときに `dis_dsp` を呼び出してもエラーに
ならず、ディスパッチ禁止状態のまま `E_OK` を返す**。`dis_dsp` と `ena_dsp` の
対応は**ネストしない**（μITRON4.0と異なり、ネストカウントを持たない）。すなわち
ディスパッチ禁止状態で `dis_dsp` を複数回呼んでも、1回の `ena_dsp` で
ディスパッチ許可状態に戻る。

ディスパッチ禁止状態では、待ち状態に入るサービスコール（`slp_tsk`・`wai_sem` 等）を
呼び出すことはできない（`E_CTX`）。

## 使用例
```c
void task_local_critical(intptr_t exinf) {
    dis_dsp();
    /* 他タスクに横取りされないが、割込みは受け付けるクリティカルセクション */
    update_task_shared_data();
    ena_dsp();
}
```

## 関連
- `ena_dsp` — ディスパッチの許可
- `sns_dsp` — ディスパッチ禁止状態の参照
- `loc_cpu` / `unl_cpu` — CPUロック / 解除（割込みも禁止するより強い排他制御）
- 仕様書 4.7 システム状態管理機能

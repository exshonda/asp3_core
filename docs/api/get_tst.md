# get_tst — タスク状態の参照（簡易）

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = get_tst(ID tskid, STAT *p_tskstat);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskid` | `ID` | 対象タスクのID番号。`TSK_SELF`（=0）で自タスク |
| `p_tskstat` | `STAT *` | タスク状態を格納する領域へのポインタ |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |
| `*p_tskstat` | `STAT` | 対象タスクの状態（下表） |

`*p_tskstat` に格納される値：

| 値 | 意味 |
|---|---|
| `TTS_RUN` | 実行状態 |
| `TTS_RDY` | 実行可能状態 |
| `TTS_WAI` | 待ち状態 |
| `TTS_SUS` | 強制待ち状態 |
| `TTS_WAS` | 二重待ち状態（待ち状態かつ強制待ち状態） |
| `TTS_DMT` | 休止状態 |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態で呼び出した |
| `E_ID` | `tskid` が不正・範囲外（`TSK_SELF` 以外を指定した場合のみチェック） |

## 機能
`tskid` で指定したタスクの状態のみを読み出し、`p_tskstat` が指す領域に格納する。
`ref_tsk` の簡易版であり、タスク状態以外の情報（優先度・待ち要因など）は取得しない。
状態の取得だけが目的の場合は、パケット構造体が不要で実行コストも低いこのコールを使う。

`TSK_SELF`（=0）を指定すると自タスクを対象とする。

## 使用例
```c
void monitor_task(EXINF exinf) {
    STAT stat;
    if (get_tst(WORKER_TASK, &stat) == E_OK) {
        if (stat == TTS_DMT) {
            act_tsk(WORKER_TASK);   /* 休止中なら起動 */
        }
    }
}
```

# ini_mpf — 固定長メモリプールの再初期化

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態でも `E_CTX`）

## C言語API
```c
ER ercd = ini_mpf(ID mpfid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mpfid` | `ID` | 対象固定長メモリプールのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態（`CHECK_TSKCTX_UNL`） |
| `E_ID` | `mpfid` が不正・範囲外 |

## 機能
`mpfid` で指定した固定長メモリプールを初期化する。

メモリプールの待ち行列を初期化し、メモリブロックの獲得を待っていたタスクの待ち状態を
解除する。また、メモリプールの管理情報を生成時の状態に戻す。具体的には、割付け可能な
空きメモリブロック数（`fblkcnt`）を生成時のブロック総数に戻し、未使用ブロックの管理
（`unused`）と空きリスト（`freelist`）を初期状態にリセットする。これにより、それまでに
割り付けていたメモリブロックの割付け状態はすべて破棄される。

待ち解除によりディスパッチが必要になればディスパッチが発生する。

## 使用例
```c
void recovery(intptr_t exinf) {
    /* 異常状態からの復旧時にメモリプールを初期状態へ戻す */
    ini_mpf(WORK_MPF);
}
```

## 関連
- `get_mpf` / `rel_mpf` — 固定長メモリブロックの獲得 / 返却
- `ref_mpf` — 固定長メモリプールの状態参照
- `CRE_MPF` — 固定長メモリプールの生成（静的API）

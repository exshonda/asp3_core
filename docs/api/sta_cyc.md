# sta_cyc — 周期通知の動作開始

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼び出すと `E_CTX`）

## C言語API
```c
ER ercd = sta_cyc(ID cycid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `cycid` | `ID` | 対象周期通知のID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態で呼び出した |
| `E_ID` | `cycid` が不正・範囲外 |

## 機能
`cycid` で指定した周期通知を動作開始状態にする。

周期通知は、まず動作開始から起動位相時間（`cycphs`、生成時に `CRE_CYC` で指定）が経過した時点で
初回の通知を行い、以降は起動周期（`cyctim`）ごとに繰り返し通知ハンドラを起動する。

対象周期通知が既に動作している場合は、それまでの起動予定をいったん解除し、この `sta_cyc` を
呼び出した時点を起点として、起動位相時間後に初回通知を行うよう設定し直す。

周期通知に `TA_STA` 属性が指定されている場合は、カーネルの起動時から動作開始状態になっているため、
`sta_cyc` を呼ばなくても起動する。

周期通知の通知ハンドラは非タスクコンテキストで実行される。ハンドラからタスクを起こすには
`wup_tsk`・`sig_sem`・`set_flg`・`psnd_dtq` など〔I〕または〔TI〕のサービスコールを使う。

## 使用例
```c
/* 周期通知の通知ハンドラ（CFGで CRE_CYC 登録） */
void sample_handler(intptr_t exinf) {
    set_flg(SYS_FLG, EV_SAMPLE);
}

void control_task(intptr_t exinf) {
    sta_cyc(SAMPLE_CYC);     /* 周期サンプリングを開始 */
    /* ... */
    stp_cyc(SAMPLE_CYC);     /* 周期サンプリングを停止 */
}
```

## 関連
- `stp_cyc` — 周期通知の動作停止
- `ref_cyc` — 周期通知の状態参照
- `sta_alm` — アラーム通知の開始（1回だけ起動）

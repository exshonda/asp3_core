# rel_mpf — 固定長メモリブロックの返却

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態でも `E_CTX`）

## C言語API
```c
ER ercd = rel_mpf(ID mpfid, void *blk);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mpfid` | `ID` | 対象固定長メモリプールのID番号 |
| `blk` | `void *` | 返却するメモリブロックの先頭番地 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（メモリブロックを返却した） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態（`CHECK_TSKCTX_UNL`） |
| `E_ID` | `mpfid` が不正・範囲外 |
| `E_PAR` | `blk` が不正（プール領域外、ブロック境界に整合しない、未割付けブロックの番地など） |

## 機能
`mpfid` で指定した固定長メモリプールに、`blk` で指定したメモリブロックを返却する。

`blk` には、`get_mpf`／`pget_mpf`／`tget_mpf` で獲得したメモリブロックの先頭番地を
指定する。番地がプール領域の先頭以降であること、ブロックサイズの境界に整合していること、
既に割り付けられた（`INDEX_ALLOC` 状態の）ブロックであることが検査され、いずれかに反する
場合は `E_PAR` を返す。

メモリプールの待ち行列にメモリブロック獲得を待っているタスクがある場合は、待ち行列の
先頭タスク（最も優先度が高く、同優先度では最も古い）に返却されたブロックを直接渡し、
その待ち状態を解除する。これによりディスパッチが必要になればディスパッチが発生する。
待ちタスクがない場合は、ブロックをプールの空きリストに戻す。

## 使用例
```c
void worker(intptr_t exinf) {
    void *blk;
    if (get_mpf(WORK_MPF, &blk) == E_OK) {
        /* blk を使った処理 */
        rel_mpf(WORK_MPF, blk);
    }
}
```

## 関連
- `get_mpf` — 固定長メモリブロックの獲得
- `pget_mpf` / `tget_mpf` — 獲得（ポーリング／タイムアウト付き）
- `CRE_MPF` — 固定長メモリプールの生成（静的API）

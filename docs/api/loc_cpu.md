# loc_cpu — CPUロック状態への遷移

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

## C言語API
```c
ER ercd = loc_cpu(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |

> エラーは返らない。既にCPUロック状態であっても `E_OK` で正常終了する（ASP3仕様）。

## 機能
CPUロック状態へ遷移させる。CPUロック状態では、すべての割込みおよびディスパッチが
禁止され、ほとんどのサービスコールが呼び出せなくなる（呼び出すと `E_CTX` を返す）。
排他制御の最も強い手段であり、ごく短い区間でのみ用いる。

ASP3では、**既にCPUロック状態のときに `loc_cpu` を呼び出してもエラーにならず、
CPUロック状態のまま `E_OK` を返す**。ただし `loc_cpu` と `unl_cpu` の対応は
**ネストしない**（μITRON4.0と異なり、ネストカウントを持たない）。すなわち
CPUロック状態で `loc_cpu` を複数回呼んでも、1回の `unl_cpu` でCPUロックは解除される。

CPUロック状態への遷移前にディスパッチ禁止状態であったかどうかは保存され、
`unl_cpu` による解除時に復元される。

## 使用例
```c
void critical_section(intptr_t exinf) {
    loc_cpu();
    /* 割込み・ディスパッチを禁止した最短のクリティカルセクション */
    shared_counter++;
    unl_cpu();
}
```

## 関連
- `unl_cpu` — CPUロック状態の解除
- `sns_loc` — CPUロック状態の参照
- `dis_dsp` / `ena_dsp` — ディスパッチの禁止 / 許可（より弱い排他制御）
- 仕様書 4.7 システム状態管理機能

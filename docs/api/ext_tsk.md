# ext_tsk — 自タスクの終了

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = ext_tsk(void);
```

## パラメータ
なし。

## リターンパラメータ
正常に終了した場合、このサービスコールから呼び出し元へ戻ることはない。
エラーの場合のみエラーコードを返す。

| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | エラーコード（正常時は戻らない） |

## エラーコード
| コード | 条件 |
|---|---|
| `E_CTX` | 非タスクコンテキストから呼び出した |
| `E_SYS` | システムエラー（正常時には返らない。タスク終了後に制御が戻った場合） |

## 機能
自タスクを終了させ、休止状態（DORMANT）に遷移させる。正常に終了した場合、このコールから
呼び出し元へ戻ることはない。

終了するタスクに対する起動要求がキューイングされている場合は、そのタスクを再度起動する
（キューイング数を1減じて実行可能状態に遷移させる）。

`ext_tsk` は次の状態でも呼び出せる。これらの状態は終了処理の中で自動的に解除される：

- CPUロック状態 → CPUロックを解除してから終了
- ディスパッチ禁止状態 → ディスパッチ許可状態にしてから終了
- 割込み優先度マスク（IPM）が `TIPM_ENAALL` 以外 → `TIPM_ENAALL` にしてから終了

非タスクコンテキストから呼び出した場合は `E_CTX` を返す。

> 注意：タスクのエントリ関数から単純に `return` した場合も `ext_tsk` を呼んだのと同じく
> タスクが終了するが、明示的に `ext_tsk` を呼ぶことが推奨される。

## 使用例
```c
void worker_task(EXINF exinf) {
    do_work();
    ext_tsk();          /* ここから先は実行されない */
}
```

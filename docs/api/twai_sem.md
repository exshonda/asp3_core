# twai_sem — セマフォ資源の獲得（タイムアウト付き）

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = twai_sem(ID semid, TMO tmout);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `semid` | `ID` | 対象セマフォのID番号 |
| `tmout` | `TMO` | タイムアウト時間（μs）。`TMO_POL`（=0）でポーリング、`TMO_FEVR`（=-1）で永久待ち |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（資源を獲得した） |
| `E_CTX` | 非タスクコンテキスト／CPUロック状態／ディスパッチ禁止状態 |
| `E_ID` | `semid` が不正・範囲外 |
| `E_PAR` | `tmout` が不正（`TMO_FEVR`・`TMO_POL` 以外の負値など） |
| `E_TMOUT` | タイムアウト時間内に資源を獲得できなかった |
| `E_RLWAI` | 待ち状態が強制解除された（`rel_wai`） |
| `E_RASTER` | タスク終了要求を受け付けている状態で呼び出した |

## 機能
`semid` で指定したセマフォから資源を1つ獲得する。`wai_sem` にタイムアウト機能を加えた
サービスコールである。

セマフォの資源数が1以上の場合は、資源数を1減じて即座に `E_OK` でリターンする。資源数が
0の場合は、自タスクをセマフォの待ち行列につなぎ、資源が返却されるか `tmout` で指定した
時間が経過するまで待ち状態に遷移する。タイムアウトすると `E_TMOUT` を返す。

`tmout` に `TMO_POL`（=0）を指定すると待ち状態に入らずポーリング動作となり（`pol_sem`
と等価）、`TMO_FEVR`（=-1）を指定するとタイムアウトを行わない（`wai_sem` と等価）。

## 使用例
```c
void worker(intptr_t exinf) {
    /* 最大10ms（10000μs）待つ */
    ER ercd = twai_sem(RESOURCE_SEM, 10000);
    if (ercd == E_OK) {
        /* クリティカルセクション */
        sig_sem(RESOURCE_SEM);
    }
    else if (ercd == E_TMOUT) {
        /* タイムアウト処理 */
    }
}
```

## 関連
- `wai_sem` — セマフォ資源の獲得（永久待ち）
- `pol_sem` — セマフォ資源の獲得（ポーリング）
- `sig_sem` — セマフォ資源の返却

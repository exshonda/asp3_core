# ini_pdq — 優先度データキューの再初期化

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = ini_pdq(ID pdqid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `pdqid` | `ID` | 対象優先度データキューのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキスト／CPUロック状態 |
| `E_ID` | `pdqid` が不正・範囲外 |

## 機能
`pdqid` で指定した優先度データキューを初期状態に戻す。

具体的には、送信待ち行列と受信待ち行列につながれていたタスクの待ちをすべて解除し
（解除されたタスクには `E_DLT` が返される）、優先度データキュー管理領域に格納されていた
データをすべて破棄する（格納データ数を0に戻し、管理領域の空きリストを初期化する）。

待ちを解除されたタスクが自タスクより高い優先度を持つ場合、本サービスコールからのリターン時に
ディスパッチが発生する。

## 使用例
```c
void reset_handler(intptr_t exinf) {
    /* キューを空にし、待ちタスクを解除して系を初期化 */
    ini_pdq(JOB_PDQ);
}
```

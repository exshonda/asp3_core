# tloc_mtx — ミューテックスのロック（タイムアウト付き）

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態・ディスパッチ禁止状態でも `E_CTX`）

## C言語API
```c
ER ercd = tloc_mtx(ID mtxid, TMO tmout);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mtxid` | `ID` | 対象ミューテックスのID番号 |
| `tmout` | `TMO` | タイムアウト時間（μs）。`TMO_POL`（=0）でポーリング、`TMO_FEVR`（=-1）で永久待ち |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（ミューテックスをロックした） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態／ディスパッチ禁止状態（`CHECK_DISPATCH`） |
| `E_ID` | `mtxid` が不正・範囲外 |
| `E_PAR` | `tmout` が不正（`VALID_TMOUT` 違反） |
| `E_ILUSE` | 優先度上限ミューテックス（`TA_CEILING`）で、自タスクのベース優先度が上限優先度より低い |
| `E_OBJ` | 自タスクが既に対象ミューテックスをロックしている（多重ロック） |
| `E_TMOUT` | タイムアウト時間内にロックを獲得できなかった（`TMO_POL` 指定で他タスクがロック中を含む） |
| `E_RASTER` | 自タスクに対してタスク終了要求がある（`ras_ter`） |
| `E_RLWAI` | 待ち状態が強制解除された（`rel_wai`） |

## 機能
`mtxid` で指定したミューテックスを、`tmout` で指定した時間を上限としてロックする。
`loc_mtx` のタイムアウト付き版である。

対象ミューテックスがロックされていなければ即座にロックを獲得して `E_OK` を返す。
他タスクがロック中の場合は、ロックが解除されるか `tmout` で指定した時間が経過するまで
待ち状態に遷移する。時間内にロックできなかった場合は `E_TMOUT` を返す。
`tmout` に `TMO_POL` を指定するとポーリング（`ploc_mtx` 相当）、`TMO_FEVR` を指定すると
永久待ち（`loc_mtx` 相当）となる。

ASP3のミューテックスは**優先度上限プロトコル（`TA_CEILING`）**のみをサポートする。
自タスクのベース優先度が上限優先度より低い場合は、ロックできず `E_ILUSE` を返す。

## 使用例
```c
void worker(intptr_t exinf) {
    ER ercd = tloc_mtx(SHARED_MTX, 1000);   /* 最大1ms待つ */
    if (ercd == E_OK) {
        /* クリティカルセクション */
        unl_mtx(SHARED_MTX);
    }
    else if (ercd == E_TMOUT) {
        /* タイムアウト処理 */
    }
}
```

## 関連
- `loc_mtx` — ロック（永久待ち）
- `ploc_mtx` — ロック（ポーリング）
- `unl_mtx` — ミューテックスのロック解除
- `CRE_MTX` — ミューテックスの生成（静的API）

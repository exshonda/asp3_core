# chg_pri — タスク優先度（ベース優先度）の変更

## 種別
〔T〕タスクコンテキスト専用（非タスクコンテキストから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = chg_pri(ID tskid, PRI tskpri);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskid` | `ID` | 対象タスクのID番号。`TSK_SELF`（=0）で自タスク |
| `tskpri` | `PRI` | 変更後のベース優先度（`TMIN_TPRI`〜`TMAX_TPRI`＝1〜16）。`TPRI_INI`（=0）で起動時優先度に戻す |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態で呼び出した |
| `E_ID` | `tskid` が不正・範囲外（`TSK_SELF` 以外を指定した場合のみチェック） |
| `E_PAR` | `tskpri` が不正・範囲外（`TPRI_INI` 以外を指定した場合のみチェック） |
| `E_OBJ` | 対象タスクが休止状態（DORMANT）である |
| `E_ILUSE` | 対象タスクが優先度上限ミューテックスをロック中／ロック待ち中で、変更後優先度がミューテックスの上限優先度より高い |

## 機能
`tskid` で指定したタスクのベース優先度を `tskpri` に変更する。`tskpri` に `TPRI_INI`（=0）を
指定すると、対象タスクの起動時優先度（生成時に指定した初期優先度）に戻す。

対象タスクがミューテックスをロックしていない場合は、ベース優先度の変更にともない
現在優先度も変更され、必要に応じてレディキュー内での順序が変更される。

対象タスクが優先度上限ミューテックスをロックしているか、その待ち状態にある場合で、
変更後のベース優先度がそのミューテックスの上限優先度よりも高い（プロトコル違反となる）
ときは `E_ILUSE` を返す。

ベース優先度の変更によって自タスクより高い優先度のタスクが実行可能になると、
ディスパッチが起こりうる。

`TSK_SELF`（=0）を指定すると自タスクを対象とする。

## 使用例
```c
void main_task(EXINF exinf) {
    /* 自タスクの優先度を一時的に上げる */
    chg_pri(TSK_SELF, HIGH_PRIORITY);
    do_critical_work();
    /* 起動時優先度に戻す */
    chg_pri(TSK_SELF, TPRI_INI);
}
```

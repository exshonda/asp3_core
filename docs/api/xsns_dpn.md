# xsns_dpn — CPU例外発生時のディスパッチ保留状態の参照

## 種別
〔I〕**CPU例外ハンドラ専用**（CPU例外ハンドラの中からのみ呼び出す）

## C言語API
```c
bool_t state = xsns_dpn(void *p_excinf);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `p_excinf` | `void *` | CPU例外ハンドラに渡されるCPU例外情報を指すポインタ |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `state` | `bool_t` | CPU例外発生時点でディスパッチ保留状態なら `true`、そうでなければ `false` |

## エラーコード
なし。

> この関数は `bool_t`（真偽値）を返し、`ER` 型のエラーコードを返さない。
> CPU例外ハンドラの中からのみ呼び出すことを想定している。

## 機能
CPU例外が発生した時点で、システムがディスパッチ保留状態であったか否かを参照する。
`sns_dpn` がCPU例外ハンドラ専用に拡張されたものであり、CPU例外ハンドラに
渡されるCPU例外情報（`p_excinf`）を用いて、**CPU例外発生時点の状態**を判定する。

以下のすべてが成り立つ場合に「ディスパッチ保留状態でない」と判定して `false` を返し、
いずれか1つでも成り立たない場合は `true` を返す。

- カーネルが動作中である（`kerflg`）
- CPU例外発生時に割込みが禁止されていない（`exc_sense_intmask(p_excinf)`）
- ディスパッチ許可状態である（`enadsp`）
- 実行中タスクが存在する（`p_runtsk != NULL`）

CPU例外ハンドラの中で、その例外を起こした処理単位がタスクコンテキストでかつ
ディスパッチ可能な状態であったかを判定し、ハンドラからタスクを起床させる等の
処理を安全に行えるかどうかを判断するために用いる。`xsns_dpn` が `false` を
返す場合に限り、CPU例外ハンドラからディスパッチを伴うサービスコールを
安全に発行できる。

## 使用例
```c
void cpu_exception_handler(void *p_excinf) {
    if (xsns_dpn(p_excinf)) {
        /* ディスパッチ保留状態：タスクを起床させず、最小限の処理に留める */
        return;
    }
    /* ディスパッチ可能：エラー処理タスクを起床できる */
    iwup_tsk(ERROR_HANDLER_TASK);
}
```

# sus_tsk — 強制待ち状態への移行

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = sus_tsk(ID tskid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskid` | `ID` | 対象タスクのID番号。`TSK_SELF`（=0）で自タスク指定 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（対象タスクを強制待ち状態に移行した） |
| `E_CTX` | 非タスクコンテキスト／CPUロック状態（`CHECK_TSKCTX_UNL`）、または自タスク指定でディスパッチ禁止状態 |
| `E_ID` | `tskid` が不正・範囲外（`TSK_SELF` 以外で `VALID_TSKID` 不成立） |
| `E_OBJ` | 対象タスクが休止状態（DORMANT） |
| `E_RASTER` | 対象タスクに終了要求がある（`ras_ter` 後） |
| `E_QOVR` | 既に強制待ち状態であり、二重には移行できない（強制待ち要求のネスト不可） |

## 機能
`tskid` で指定したタスクを強制待ち状態（SUSPENDED）に移行させる。

- 対象タスクが実行できる状態（RUNNABLE）の場合、強制待ち状態（SUSPENDED）へ遷移させる。
  自タスクを指定して実行できる状態から強制待ちに入る場合は、ディスパッチが起こる。
- 対象タスクが待ち状態（WAITING）の場合、二重待ち状態（WAITING かつ SUSPENDED）へ遷移させる。
- 対象タスクが既に強制待ち状態の場合は、強制待ち要求はネストできないため `E_QOVR` を返す。

ASP3では強制待ち要求のキューイングはサポートされず、強制待ち状態の多重化（カウントアップ）は
行わない。

自タスク（`TSK_SELF` または自タスクのID）を指定する場合、ディスパッチ禁止状態では `E_CTX` を返す
（自タスクが強制待ちに入るとディスパッチが必要になるため）。

## 使用例
```c
void supervisor_task(intptr_t exinf) {
    /* ワーカタスクを一時停止する */
    ER ercd = sus_tsk(WORKER_TASK);
    if (ercd == E_OK) {
        adjust_parameters();
        rsm_tsk(WORKER_TASK);   /* 再開 */
    }
}
```

## 関連
- `rsm_tsk` — 強制待ち状態からの再開（`sus_tsk` の対） 
- `ter_tsk` — タスクの強制終了

# loc_mtx — ミューテックスのロック

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態・ディスパッチ禁止状態でも `E_CTX`）

## C言語API
```c
ER ercd = loc_mtx(ID mtxid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mtxid` | `ID` | 対象ミューテックスのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（ミューテックスをロックした） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態／ディスパッチ禁止状態（`CHECK_DISPATCH`） |
| `E_ID` | `mtxid` が不正・範囲外 |
| `E_ILUSE` | 優先度上限ミューテックス（`TA_CEILING`）で、自タスクのベース優先度が上限優先度より低い |
| `E_OBJ` | 自タスクが既に対象ミューテックスをロックしている（多重ロック） |
| `E_RASTER` | 自タスクに対してタスク終了要求がある（`ras_ter`） |
| `E_RLWAI` | 待ち状態が強制解除された（`rel_wai`） |

## 機能
`mtxid` で指定したミューテックスをロックする。対象ミューテックスがどのタスクにも
ロックされていない場合は、自タスクがロックを獲得して即座にリターンする。
既に他タスクがロックしている場合は、自タスクをミューテックスの待ち行列につなぎ、
ロックが解除される（`unl_mtx`）まで待ち状態に遷移する。

ASP3のミューテックスは**優先度上限プロトコル（`TA_CEILING`）**のみをサポートし、
優先度継承プロトコルはサポートしない。優先度上限ミューテックスをロックすると、
自タスクの現在優先度はミューテックスの上限優先度まで引き上げられる（優先度上昇状態）。
このため、自タスクのベース優先度が上限優先度より低い（数値が大きい）場合はロックできず、
`E_ILUSE` を返す。

ポーリングを行う場合は `ploc_mtx`、タイムアウトを設ける場合は `tloc_mtx` を使う。

## 使用例
```c
void worker(intptr_t exinf) {
    ER ercd = loc_mtx(SHARED_MTX);
    if (ercd == E_OK) {
        /* クリティカルセクション */
        unl_mtx(SHARED_MTX);
    }
}
```

## 関連
- `unl_mtx` — ミューテックスのロック解除
- `ploc_mtx` — ロック（ポーリング）
- `tloc_mtx` — ロック（タイムアウト付き）
- `CRE_MTX` — ミューテックスの生成（静的API）

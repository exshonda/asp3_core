# sns_ctx — コンテキストの参照

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能（CPUロック状態でも呼出し可能）

## C言語API
```c
bool_t state = sns_ctx(void);
```

## パラメータ
なし

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `state` | `bool_t` | 非タスクコンテキストなら `true`、タスクコンテキストなら `false` |

## エラーコード
なし。

> この関数は `bool_t`（真偽値）を返し、`ER` 型のエラーコードを返さない。
> どのようなコンテキスト・状態（CPUロック状態を含む）からでも呼び出せ、エラーにならない。

## 機能
現在の実行コンテキストが非タスクコンテキストか否かを参照する。
非タスクコンテキスト（割込みハンドラ・CPU例外ハンドラ・周期通知/アラーム通知の
通知ハンドラなど）から呼び出した場合は `true`、タスクコンテキストから呼び出した
場合は `false` を返す。

状態を参照するだけで、カーネルの状態を変化させない。CPUロック状態であっても
呼び出すことができ、エラーにならない。

## 使用例
```c
/* タスク・非タスク両方から呼ばれる共通ルーチン内で分岐する */
void common_routine(void) {
    if (sns_ctx()) {
        /* 非タスクコンテキスト：iwup_tsk 等を使う */
        iwup_tsk(MAIN_TASK);
    } else {
        /* タスクコンテキスト */
        wup_tsk(MAIN_TASK);
    }
}
```

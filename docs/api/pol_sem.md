# pol_sem — セマフォ資源の獲得（ポーリング）

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`）

## C言語API
```c
ER ercd = pol_sem(ID semid);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `semid` | `ID` | 対象セマフォのID番号 |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（資源を獲得した） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態 |
| `E_ID` | `semid` が不正・範囲外 |
| `E_TMOUT` | ポーリング失敗（資源数が0で獲得できなかった） |

## 機能
`semid` で指定したセマフォから資源を1つ獲得する（ポーリング）。`wai_sem` と異なり、
資源を獲得できない場合でも待ち状態には遷移しない。

セマフォの資源数が1以上の場合は、資源数を1減じて `E_OK` でリターンする。資源数が0の
場合は、待ち状態に入らず即座に `E_TMOUT` を返す。

`pol_sem` は、`tmout` に `TMO_POL` を指定した `twai_sem` と等価である。

## 使用例
```c
void worker(intptr_t exinf) {
    ER ercd = pol_sem(RESOURCE_SEM);
    if (ercd == E_OK) {
        /* クリティカルセクション */
        sig_sem(RESOURCE_SEM);
    }
    else if (ercd == E_TMOUT) {
        /* 資源が空いていなかった。別の処理へ */
    }
}
```

## 関連
- `wai_sem` — セマフォ資源の獲得（ブロッキング）
- `twai_sem` — セマフォ資源の獲得（タイムアウト付き）
- `sig_sem` — セマフォ資源の返却

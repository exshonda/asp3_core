# get_mpf — 固定長メモリブロックの獲得

## 種別
〔T〕タスクコンテキスト専用（非タスクから呼ぶと `E_CTX`／CPUロック状態・ディスパッチ禁止状態でも `E_CTX`）

## C言語API
```c
void *blk;
ER ercd = get_mpf(ID mpfid, void **p_blk);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `mpfid` | `ID` | 対象固定長メモリプールのID番号 |
| `p_blk` | `void **` | 獲得したメモリブロックの先頭番地を格納する変数へのポインタ |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |
| `*p_blk` | `void *` | 獲得したメモリブロックの先頭番地 |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了（メモリブロックを獲得した） |
| `E_CTX` | 非タスクコンテキストから呼び出した／CPUロック状態／ディスパッチ禁止状態（`CHECK_DISPATCH`） |
| `E_ID` | `mpfid` が不正・範囲外 |
| `E_RASTER` | 自タスクに対してタスク終了要求がある（`ras_ter`） |
| `E_RLWAI` | 待ち状態が強制解除された（`rel_wai`） |

## 機能
`mpfid` で指定した固定長メモリプールから固定長メモリブロックを1つ獲得し、その先頭番地を
`*p_blk` に格納する。プールに割付け可能なメモリブロックがある場合は、それを獲得して
即座にリターンする。割付け可能なメモリブロックがない場合は、自タスクをメモリプールの
待ち行列につなぎ、メモリブロックが返却される（`rel_mpf`）まで待ち状態に遷移する。

獲得したメモリブロックのサイズは、メモリプール生成時（`CRE_MPF`）に指定したブロックサイズ
である。獲得したブロックの内容は不定で、使い終わったら `rel_mpf` で返却する。
カーネル内の動的メモリ確保（`malloc` 等）の代替として用いる静的なメモリ管理機構である。

ポーリングを行う場合は `pget_mpf`、タイムアウトを設ける場合は `tget_mpf` を使う。

## 使用例
```c
void worker(intptr_t exinf) {
    void *blk;
    if (get_mpf(WORK_MPF, &blk) == E_OK) {
        /* blk を使った処理 */
        rel_mpf(WORK_MPF, blk);
    }
}
```

## 関連
- `rel_mpf` — 固定長メモリブロックの返却
- `pget_mpf` — 獲得（ポーリング）
- `tget_mpf` — 獲得（タイムアウト付き）
- `CRE_MPF` — 固定長メモリプールの生成（静的API）

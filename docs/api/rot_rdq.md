# rot_rdq — タスクの優先順位の回転

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

## C言語API
```c
ER ercd = rot_rdq(PRI tskpri);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `tskpri` | `PRI` | 優先順位を回転させる対象の優先度（`TPRI_SELF`（＝0）を指定すると自タスクのベース優先度） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_CTX` | CPUロック状態で呼び出した（`CHECK_UNL`） |
| `E_PAR` | `tskpri` が優先度の範囲（`TMIN_TPRI`〜`TMAX_TPRI`）外（`CHECK_PAR(VALID_TPRI)`。ただし `TPRI_SELF` を非タスクコンテキストから指定した場合も範囲チェックの対象） |

## 機能
`tskpri` で指定した優先度のレディキュー（実行可能状態のタスクをつなぐFIFOキュー）を
回転させる。具体的には、その優先度を持つ実行可能状態のタスクのうち、
キューの先頭にあるタスクをキューの末尾に移動する。これにより、
同一優先度のタスク間でのラウンドロビンスケジューリングを実現できる。

`tskpri` に `TPRI_SELF`（＝0）を指定し、かつタスクコンテキストから呼び出した場合は、
自タスクのベース優先度を対象とする。非タスクコンテキストから `TPRI_SELF` を
指定することはできない（優先度範囲チェックで `E_PAR` となる）。

回転の結果、最高優先順位のタスクが変化した場合は、CPUロック解除と同時に
ディスパッチが起こる（タスクコンテキストの場合）。

## 使用例
```c
/* 周期ハンドラで同一優先度タスクを時分割実行する */
void rotate_handler(intptr_t exinf) {
    rot_rdq(MID_PRIORITY);   /* 中優先度のレディキューを回転 */
}
```

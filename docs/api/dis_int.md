# dis_int — 割込みの禁止

## 種別
〔TI〕タスクコンテキスト・非タスクコンテキストの両方から呼出し可能

> **ターゲット定義機能。** `dis_int` がサポートされるかどうかはターゲット依存
> （ターゲット定義マクロ `TOPPERS_SUPPORT_DIS_INT` が定義されている場合のみ提供される）。
> サポートされないターゲットでは呼び出せない。

## C言語API
```c
ER ercd = dis_int(INTNO intno);
```

## パラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `intno` | `INTNO` | 禁止対象の割込み番号（プロセッサ依存） |

## リターンパラメータ
| 名称 | 型 | 説明 |
|---|---|---|
| `ercd` | `ER` | 正常終了（`E_OK`）またはエラーコード |

## エラーコード
| コード | 条件 |
|---|---|
| `E_OK` | 正常終了 |
| `E_PAR` | `intno` が割込み番号として不正・範囲外 |
| `E_OBJ` | `intno` が `CFG_INT` で設定されていない割込み番号 |

> 本サービスコールがサポートされていないターゲットでは、呼び出せない（プロトタイプが提供されない）。

## 機能
`intno` で指定した割込み番号の割込みを禁止する。

割込みを禁止すると、その割込み番号に対応する割込み要求は受け付けられなくなる。
禁止された割込みは `ena_int` で再び許可できる。

`intno` は `CFG_INT` で設定されている割込み番号でなければならない。設定されていない
割込み番号を指定すると `E_OBJ` を返す。

非タスクコンテキストから呼び出す場合は、同一機能の `idis_int` を使ってもよい
（ASP3では `idis_int` は `dis_int` のマクロとして定義されている）。

## 使用例
```c
void critical_section(void) {
    dis_int(INTNO_UART);    /* UART割込みを禁止 */
    /* 割込みと競合する処理 */
    ena_int(INTNO_UART);    /* UART割込みを許可 */
}
```

## 関連
- `ena_int` — 割込みの許可（対操作）
- `clr_int` / `ras_int` / `prb_int` — 割込み要求のクリア / 要求 / チェック（ターゲット定義機能）
- [CFG_INT](CFG_INT.md) — 割込み要求ラインの設定（`intno` はここで設定したもの）
